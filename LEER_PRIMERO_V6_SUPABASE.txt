NATURA VIDA V6 — CORRECCIÓN ONLINE-FIRST

Cambios aplicados:
1. Supabase Auth queda como mecanismo oficial de ingreso.
2. Se mantiene ingreso por Gmail + contraseña.
3. Se mantiene creación de cuenta:
   - primer usuario o código 27121961 = administrador activo.
   - usuario normal = representante pendiente hasta aprobación.
4. Se agregó recuperación completa de contraseña:
   - Olvidé mi contraseña envía correo desde Supabase.
   - El enlace de recuperación permite definir nueva contraseña dentro de la app.
5. IndexedDB ya no se usa como identidad ni fuente principal.
   Sólo queda para caché temporal, cola offline y respaldo auxiliar.
6. Se actualizó el texto de la app para no decir offline-first como modelo principal.
7. Se agregó SQL maestro: SUPABASE_V6_ONLINE_FIRST.sql

PASOS OBLIGATORIOS:
1. En Supabase, ejecutar SUPABASE_V6_ONLINE_FIRST.sql en SQL Editor.
2. En Authentication > Providers, confirmar Email habilitado.
3. En Authentication > URL Configuration colocar:
   Site URL: el enlace de GitHub Pages de la app.
   Redirect URLs: el mismo enlace con /* al final.
4. Editar js/supabase-config.js y pegar Project URL y anon public key.
5. Subir todos estos archivos a la raíz del repositorio GitHub Pages.
6. Abrir la app y crear primero la cuenta administradora.

IMPORTANTE:
No usar copias JSON como base principal del negocio. En V6 la verdad oficial debe estar en Supabase.
